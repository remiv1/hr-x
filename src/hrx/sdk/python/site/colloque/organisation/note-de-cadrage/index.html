Note de Cadrage — HR-X

Standard ouvert de la donnée candidat.

Colloque Fondateur

1. Constat partagé

Le processus de recrutement repose aujourd’hui sur un artefact obsolète : le CV PDF, un document aplati, non structuré, difficilement exploitable.

Les ATS tentent de reconstituer la donnée via du parsing, ce qui entraîne :

  • perte de qualité,
  • erreurs d’interprétation,
  • asymétrie d’accès entre candidats,
  • biais involontaires,
  • inefficacité opérationnelle,
  • génération d'un aléat (coïncidence entre format et ATS).

Ce problème est systémique et touche l’ensemble des acteurs : candidats, entreprises, éditeurs, institutions.

2. L’ambition HR‑X

HR‑X vise à créer un standard ouvert, inspiré de Factur‑X, permettant :

  • de conserver la donnée dans un format structuré,
  • d’assurer l’équité et l’inclusion,
  • de faciliter l’intégration dans les architectures hexagonales,
  • de réduire les biais liés au parsing,
  • d’améliorer la qualité des données RH,
  • de sortir du cercle vicieux où chaque candidat envoie toujours plus de CV pour augmenter les chances.

HR‑X n’est pas un produit, mais un standard interopérable, souverain et neutre.

3. Trois briques fondatrices

1. Plateforme de génération de données pour les candidats

Un outil simple permettant de produire un document hybride : PDF lisible + données structurées embarquées (JSON ou XML).

→ Plateforme HR‑X

2. Sandbox de test pour les entreprises et éditeurs

Un environnement permettant de valider la conformité HR‑X, tester l’intégration et garantir l’interopérabilité.

→ Sandbox HR‑X

3. SDK / JDK de parsing HR‑X

Des bibliothèques prêtes à l’emploi, intégrables dans les architectures hexagonales (ports/adapters).

→ SDK HR‑X

Ces trois briques suffisent pour lancer un standard fonctionnel et testable et surtout facilement assimilable par les différents acteurs du marché.

4. Le colloque fondateur (visio)

Le colloque fondateur est une réunion en visioconférence de 45 minutes, réunissant :

  • 3 à 5 grands groupes,
  • des représentants ESR/RSE, RH, innovation ou communication corporate,
  • un coordinateur neutre (auditeur indépendant).

Objectifs :

  • partager le constat,
  • valider l’intérêt du standard,
  • aligner les acteurs sur les trois briques,
  • définir les premiers cas d’usage,
  • signer un engagement d’exploration non contraignant.

→ Engagement d’exploration

Aucun engagement financier n’est demandé.

5. Rôle des acteurs fondateurs

Les acteurs fondateurs :

  • participent à la définition du standard,
  • apportent leur expertise métier, RH et technique,
  • identifient les contraintes internes,
  • contribuent à la validation des prototypes,
  • deviennent les premiers promoteurs d’un standard souverain,
  • deviennent membres permanents de la propriété du standard.

Ils ne développent rien : ils orientent, valident, légitiment.
